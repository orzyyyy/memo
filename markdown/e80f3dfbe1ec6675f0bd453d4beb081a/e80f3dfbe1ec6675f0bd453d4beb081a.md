<h2 id="尾数">尾数</h2>
<ol>
<li><p>在四个选项中看出要算几位尾数</p>
<p>如 1.1、2.2、3.3、4.4，需要算一位尾数</p>
<p>如 11.1、12.1、13.1、14.1，需要算两位尾数</p>
</li>
<li><p>算</p>
<p>比如 1692.4 + 5536.8 + x = 8123.2</p>
<p>选项为 894.0、894.6、895.2、896.1</p>
<p>此时只需要算一位尾数，即 x = 2 - 8 - 4</p>
<p>2 来自 8123.2，8 来自 5536.8，4 来自 1692.4</p>
<p>2 - 8 - 4 不足，则向前借 10（借位时需要借到结果为正数），12 - 8 - 4 = 0，所以答案是 894.0</p>
<hr>
<p>现在把选项改为 894.0、893.0、895.0、896.0</p>
<p>此时需要算两位尾数，即 x = 32 - 68 - 24</p>
<p>32 来自 8123.2，68 来自 5536.8，24 来自 1692.4</p>
<p>32 - (68 + 24) = 32 - 92，尾数不足，需要借到结果为正，于是借 100</p>
<p>于是算式变为 132 - 92 = 40，于是答案为后两位为 40 的 894.0</p>
</li>
</ol>
<h2 id="从高位算起的加法">从高位算起的加法</h2>
<p>比如 894.0 + 894.6 + 895.2 + 896.1</p>
<p>传统算法为从个位数开始累加、进位，得到结果</p>
<p>该算法从最高位开始加，如</p>
<p>8 9 4 . 0</p>
<p>8 9 4 . 6</p>
<p>8 9 5 . 2</p>
<p>8 9 6 . 1</p>
<p>第一位四个 8 相加，结果为 32</p>
<p>第二位四个 9 相加，结果为 36</p>
<p>第三位 4 + 4 + 5 + 6，结果为 19</p>
<p>第四位 0 + 6 + 2 + 1，结果为 9</p>
<pre><code class="language-bash">3 2
  3 6
    1 9
      0 9
——————————————
3 5 7 9 9</code></pre>
<p>加上小数点也就是 3579.9</p>
<ul>
<li><p>列竖式时需要右对齐，这样可以解决相加项不同位数的问题</p>
</li>
<li><p>竖式某行如果和小于 10，也就是结果只有一位时，需要在前面补上 0。比如上方第四行的 9，只有一位，需要在竖式中写成 09 来对齐</p>
</li>
</ul>
<h2 id="拆分">拆分</h2>
<ol>
<li><p>除法的拆分</p>
<p>比如 685 / 752</p>
<p>估一下 685 大致占 752 的 90% 多。如果直接算 752 * 0.9 会比较复杂，可以写成 1 - x 的形式</p>
<p>于是可以变式为 1 - (752 - 685) / 752 = 1 - 67 / 752</p>
<p>继续拆分，67 占 752 不到 10%，大致可以推算结果比 90% 大一些</p>
<p>可以继续变式为 (75.2 - 8.2) / 752</p>
<p>最终变式为 1 - (0.1 - 8.2 / 752) = 1 - 0.1 + 8.2 / 752</p>
<p>估一下 8.2 占 752 大致比 1% 大一些，所以最后得到的数大致为 91% 大一些，且靠近 91%</p>
<p>如果需要更精确的值，后面以此类推</p>
</li>
<li><p>乘法的拆分</p>
<p>比如 789 * 147</p>
<p>观察因子，147 有效数字接近 1 / 7 = 14.2%</p>
<p>于是变式为 789 / 7 = 112，结果有效数字大致是以 112 开头的</p>
<p>实际结果为 115983，所以该方法只能在选项差距较大时使用</p>
</li>
</ol>
<h2 id="比较分数大小">比较分数大小</h2>
<p> 比如 48 / 92 和 72 / 136</p>
<p> 此处有个知识前提，分子变化量大于分母变化量时，分数整体变大，否则变小</p>
<p> 上式分子变化量为 (72 - 48) / 48 = 50%，分母变化量 (136 - 92) / 92 &lt; 50%</p>
<p> 于是分数整体变大，于是前者小于后者</p>
<p> 如果分子分母为多位数，比如 123456 / 1.2975 和 654321/1.67812</p>
<p> 这时保留三个有效数字即可，误差一般不会太大</p>
<p> 上面的例子估值为 123 / 1.30 * 1000 = 95348.837，计算值为 123456 / 1.2975 = 95149.132</p>
<h2 id="已知现期和变化率，同时求基期和变化量">已知现期和变化率，同时求基期和变化量</h2>
<p>比如现期 = 1360，变化率 = 23%，求基期和变化量</p>
<p>传统的算法，基期 = 1360 / (1 + 23%)，变化量 = 1360 * 23%</p>
<p>现在的算法：</p>
<pre><code class="language-bash">           1360
             |
             |
       ——————————————
       |     |      |
       |     |      |
     1000    | 1000 * 23% = 230
             |
             |
   1360 - (1000 + 230) = 130
             |
             |
       ——————————————
       |            |
       |            |
      100           23</code></pre>
<p>一般计算到这一步就可以确定答案了，如果需要更精确的数字，下面步骤以此类推</p>
<p>基期为竖式左侧相加 1000 + 100 = 1100 左右，变化量为竖式右侧相加 230 + 23 = 253 左右</p>
<h2 id="隔年增长率">隔年增长率</h2>
<p>设今年增长 R1，去年增长 R2，则前年增长为 R1 + R2 + R1 * R2</p>
<h2 id="比值增长率">比值增长率</h2>
<p>R = (R1 - R2) / (1 + R2)</p>
<p>比如单位面积平均售价增速 =&gt; 求今年单位面积平均售价比去年的增长率</p>
<p>单位面积是 总价 / 总面积，于是 R1 为总价，R2 为总面积</p>
<h2 id="比重差">比重差</h2>
<p>R = a / B * (R1 - R2)</p>
<p>a 为基期的一部分，B 为现期整体，R1 为现期部分增长率，R2 为现期整体增长率</p>
<p>比如 今年总额为 B，同比增长 R2。B 今年的一部分为 a1，同比增长为 R1。求 a1 今年占 B 的比重和上年同期比重之差</p>
<p>去年的部分 a = a1 / (1 + R1)，之后即可套公式</p>
<p>一般而言，今年比去年增长时，则去年的部分一定比今年的整体小，所以 a / B 小于 1。小于 1 的数乘比重之差，所以比重差小于增长率绝对值之差</p>
<p>如果今年比去年减少，只要今年整体没减少到比去年前期还小，那就还成立</p>
<p>根据公式，a / B 一定是一个正数，所以 R1 比 R2 大时，今年比重比去年的趋势为增加。换句话说，比重差会比 R1 - R2 的绝对值小</p>
<p>更进一步，如果增长率之差较小，比如不到百分之五，则前期部分的值——也就是 a ——的估算容错率会很大。因为误差被增长率之差缩小，所以可以大胆估算</p>
<h2 id="混合问题">混合问题</h2>
<h3 id="十字相减">十字相减</h3>
<p>当最后的算式可以被转化成 A = B / C 的形式，A 中只包含 B 和 C，且上述 A, B, C 已知、求的是 C 之比时，才能用这个方法</p>
<p>确定可以用以后，把 A, B, C 放到相应位置即可</p>
<p>比如 人均收入为 x，其中一部分人均收入为 y，剩余部分人均收入为 z，求两部分人数之比</p>
<p>由于 人均收入 = 收入 / 人数，现已知两部分人均收入和总体的人均收入，于是用十字相减</p>
<pre><code class="language-bash">    y   z - x
     \   /
      \ /
       x
      / \
     /   \
    z   x - y</code></pre>
<p>于是得到式子 y / z = (z - x) / (x - y)</p>
<p>最终人数之比，也就是 C 之比为 (z - x) / (x - y)</p>
<p>实际上 B 是没被用到的</p>
<h2 id="标段">标段</h2>
<p>和十字相减不同，十字相减求的是部分之比，需要知道三个量，然后求第四个量</p>
<p>标段则是用来求这三个量中的其中一个，换句话就是已知两个量，求另一个量的方法</p>
<p>比如 整体中，其中一部分同比增长为 R1 = 2 / 3，量为 A = 10。剩余部分同比增长为 R2 = 1 / 3，量为 B = 5。求整体的同比增长 R</p>
<ol>
<li><p>先画一根数轴——R1 和 R2 为了计算方便直接假设成分数，实际情况需要化成分数</p>
<p>上述题目中比例为 1 : 3，于是需要分成四份</p>
</li>
</ol>
<pre><code class="language-bash">|          |           |           |           |
------------------------------------------------</code></pre>
<ol start="2">
<li><p>标上已知增长率</p>
<p>两端为部分增长率，中间为整体增长率</p>
<p>部分增长率从左往右，从大到小标整体增长率靠近量大的一侧</p>
<p>整体增长率靠近量大的一侧</p>
<p>上述题目中，A 较大，于是整体增长率靠近 R1</p>
</li>
</ol>
<pre><code class="language-bash"> |          |           |           |           |
 ------------------------------------------------
33.3%                                         66.7%

<p> R2                                 R           R1</code></pre></p>
<ol start="3">
<li><p>算</p>
<p>上方数轴中，R 和 R1 差了数轴中的一份，设为 x，所以只要算出 x，再算出 66.7 - x 即为答案</p>
<p>x = (66.7 - 33.3) / 4 = 8.33</p>
<p>R = 66.7 - 8.33 = 58.37</p>
</li>
</ol>
<h2 id="年均增长率">年均增长率</h2>
<p>末年量 x / 首年量 y = (1 + R) ^ n</p>
<p>比如 2003 - 2007 年间，03 年 item 量为 i1， 04 年为 i2，05 年为 i3,06 年为 i4,07 年为 i5。问 item 的年均增长率 R 为多少</p>
<p>根据公式可得 (1 + R) ^ 4 = i5 / i1</p>
<p>i5 / i1 的值只能靠猜。如果是不规则的数字，建议放弃</p>
<p>1.1 ^ 4 = 1.46，1.15 ^ 4 = 1.75，1.2 ^ 4 = 2.07</p>
<p>如果是五年的增长率，可以自行再乘一次</p>
<p>上述题目中，如果 i5 / i1 = 1.8，则 0.15 &lt; R &lt; 0.2</p>
